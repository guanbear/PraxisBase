# Deep Sleep Memory Consolidation Procedure

## Problem / Context
Automated memory maintenance is periodically required to address and repair degraded or corrupted recall artifacts. Additionally, the system must evaluate temporary short-term memories to determine if they warrant promotion to permanent, durable storage.

## Fix / Decision
The agent executes a "Deep Sleep" consolidation cycle. The primary actions decided upon are:
1. **Repair:** Rewrite the recall store to fix corrupted or degraded recall artifacts.
2. **Evaluate:** Rank and evaluate a pool of candidate memories (e.g., 4 candidates).
3. **Promote:** Apply promotion criteria to move validated memories to `MEMORY.md`.

## Verification
- Confirm that the recall store rewrite completes without error.
- Validate that the ranking and promotion pipeline correctly evaluates the candidate pool.
- *Note: In the captured instance, verification was marked as unknown, and 0 candidates were promoted.*

## Risks
- **Overly Strict Criteria:** Promoting zero candidates during a cycle might indicate that the promotion criteria are too strict, potentially causing significant short-term memories to be lost.
- **Data Loss:** Rewriting the recall store to repair artifacts carries an inherent risk of data loss if the rewrite process is interrupted or flawed.
