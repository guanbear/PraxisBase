# M12 and M12.1 Acceptance

Date: 2026-05-20
Branch: `feat/multi-agent-experience-layer`

## Scope

M12 adds local agent memory scan, ingest, and real wiki smoke for Codex and OpenClaw sources.

M12.1 adds the PraxisBase CLI control surface for non-local OpenClaw memory:

- `praxisbase memory fetch --agent openclaw --provider exported-json --source <file> --json`
- `praxisbase memory fetch --agent openclaw --provider openclaw-api --remote <id> --json`
- `praxisbase memory ingest --agent openclaw --source .praxisbase/staging/openclaw --write --json`
- `praxisbase doctor openclaw-remote --provider <provider> --json`

Fetch is a staging step only. It writes safe envelopes and reports, and it never promotes stable knowledge. Ingest writes protocol evidence only. Wiki changes still require the normal proposal, review, and promote flow.

## Implemented

- Agent memory scan and ingest for Codex session-like files and OpenClaw logs.
- Explicit `~/...` source path expansion.
- `.json`, `.jsonl`, `.md`, `.txt`, and `.log` source support.
- Per-file size cap and skipped-file reporting.
- Deterministic `source_hash` and stable `source_ref` generation.
- Raw-vault ref and capture evidence generation in write mode.
- Deduplication across raw-vault refs, capture artifacts, and memory reports.
- Private material routing to human-required exceptions.
- `smoke real-wiki` pipeline: ingest, wiki compile review, graph, site, and context.
- OpenClaw remote `exported-json` provider.
- OpenClaw remote `openclaw-api` provider with environment token support.
- Optional `openclaw-cli` provider diagnostics stub.
- Safe `.praxisbase/staging/openclaw/*.json` envelopes.
- Fetch reports under `.praxisbase/reports/memory-fetch/`.
- Fetch run records under `.praxisbase/runs/memory-fetch/`.
- Doctor diagnostics for token, base URL, OpenClaw CLI provider, and staging gitignore coverage.
- `praxisbase init` creates remote-memory directories and ignores `.praxisbase/staging/`.
- OpenClaw staged envelopes are compatible with `memory ingest`.

## Safety Checks

- `memory fetch`, `memory ingest`, and `doctor openclaw-remote` do not write `kb/` or `skills/`.
- Staged remote envelopes do not include `raw_log`, raw response bodies, auth headers, cookies, tokens, or private keys.
- Fetch and ingest reports use `changed_stable_knowledge: false`.
- `.praxisbase/staging/` is ignored by Git in this repo and in new initialized workspaces.

## Known Limits

- `openclaw-cli` is diagnostic-only in M12.1; the default supported paths are `exported-json` and `openclaw-api`.
- Current real wiki smoke validates the end-to-end pipeline and stable-knowledge safety. It does not yet score wiki quality, recall, duplication, or page usefulness against a curated real dataset.
- The next milestone should focus on real-data evaluation and generated HTML/wiki usability.
