## Smoke Findings

- The run was intentionally degraded and did not call the LLM lesson extractor: `ai_distill.mode=disabled`, `lessons.ai_lessons=0`, and `ai_distill.warnings` includes `ai_distill_disabled`.
- Source coverage was real and broad: the lesson report selected 329 spans from 871 source items, including local OpenClaw, Codex, and trusted remote OpenClaw staged material.
- The lesson layer produced 22 deterministic lessons before dedupe/state routing. The final visible states in the daily report were `active_personal=1`, `wiki_ready=0`, `skill_ready=2`, `human_required=0`, and `rejected=0`.
- The useful visible lessons were:
  - private-route remote access as `active_personal`;
  - Slack raw user id boundary/redaction as `skill_ready`;
  - ACK before slow tool/network/dispatch work as `skill_ready`.
- Wiki proposal candidates were generated from lesson evidence and include M25 sections for applicability, negative cases, agent use, related pages, and span provenance.
- The generated HTML site exposes M25 lesson counts, top lesson summaries, pending wiki candidates, pending skill candidates, review reasons, and graph links. `dist/graph.json` contains 8 nodes, 29 links, and no broken links/orphans/duplicates.
- Stable PB pages are visible in graph/site source ids through `stable_kb:*` entries. GBrain and AgentMemory sidecars were not promotion evidence in this run; the authority contract explicitly ranks them below stable PB pages, promoted skills, and active personal lessons.

## Quality Gaps Before Full Daily

- This smoke does not prove production LLM lesson extraction quality because it used `--no-ai`. A follow-up AI smoke must run with a finite uncached budget and verify cache reuse.
- `wiki_ready=0` in this run. The wiki candidates were built from `skill_ready` lesson evidence and stable pages, not from a final wiki-ready lesson state.
- AgentMemory health failed with `agentmemory_health_failed: fetch failed`, so AgentMemory sidecar import/export behavior was not exercised.
- GBrain sidecar/export behavior was not exercised because there were no changed stable PB pages and no publish path was requested.
- Skill candidates still require human review. One candidate had `skill_shape_invalid:malformed_procedure_heading`; another had semantic review `revise` with a truncated procedure sentence.
- The review queue remains large because privacy triage reported 58 human-required exceptions. This is expected for private runtime material, but it blocks some potentially useful evidence until triaged.
- The current span provenance is useful for audit, but review surfaces must keep raw private excerpts hidden by default and should avoid making noisy candidate fragments the first user-facing content.

## Outcome

M25 real smoke is acceptable as a bounded degraded smoke: it proves memory-first ingestion, deterministic lesson routing, wiki/skill candidate generation, HTML visibility, privacy gating, and authority ranking. It is not sufficient as the final production quality proof. The next validation should be a small AI-enabled daily run with cached GLM extraction and sidecar health checks.

## Quality Gate Follow-Up

Additional validation found that deterministic M25 extraction and the lesson-state cache could still preserve raw OpenClaw candidate-corpus noise. The bad signature looked like repeated `Candidate:` transcript fragments with `confidence: 0.00`, `.dreams/session-corpus`, staged candidate status, or untrusted conversation metadata. This is not a reusable lesson and must not become active personal context, skill input, or wiki evidence.

Fixes:

- Deterministic lesson extraction now skips raw candidate-corpus spans before pattern matching.
- Lesson-state cache loading/upsert now drops stale raw candidate-corpus records so previously cached bad lessons cannot keep resurfacing.
- Daily summary now carries the lesson authority contract from the lesson report; when lesson-derived wiki evidence exists, the daily report exposes `wiki_semantic_input=lesson_clusters`.

Verification:

- Focused tests passed for deterministic extraction, lesson cache cleanup, and M25 production integration.
- A real local OpenClaw `lesson extract --max-spans 80` no longer returns the old `Candidate: route=direct ... confidence: 0.00` active lesson.
- A bounded no-AI daily run produced `deterministic_lessons=12`, `skill_ready=1`, `wiki_evidence=1`, and `authority_contract.wiki_semantic_input=lesson_clusters` without spending GLM tokens.

Remaining gaps:

- AgentMemory is configured but the local daemon is not installed/running (`agentmemory not found`, daemon fetch failed).
- Trusted remote OpenClaw still produces privacy-required records on each daily run; safe abstraction and triage reuse need another pass before remote memory can flow smoothly into stable wiki/skill outputs.
- Many Codex/OpenClaw session files are oversize and currently enter only as bounded samples; better session pre-summarization/selection is still needed for full personal coverage without large token spend.

## Remote Noise Follow-Up

Additional no-AI validation showed that trusted remote OpenClaw `latest.json` can already contain `source_ref` values under `memory/dreaming/` and raw `Candidate:` corpus fragments. The earlier SQLite query filter was not enough because those JSON export items entered `resolveExperienceSource` before lesson extraction or privacy triage.

Fixes:

- The OpenClaw source adapter now excludes `memory/dreaming/`, `.dreams`, and `dream-diary` items before envelope creation and privacy triage.
- The local OpenClaw SQLite adapter uses the same path exclusions as source inventory, so dream rows do not become source envelopes.
- Candidate-corpus filtering checks each parsed item first; one noisy JSON item no longer causes every item in the same export payload to be skipped.

Verification:

- Source-adapter tests cover local SQLite dream chunks and trusted remote OpenClaw exported JSON dream items.
- `pnpm test -- tests/core/experience-source-adapters.test.ts` passed.
- A bounded real no-AI daily run completed without GLM calls. Trusted remote OpenClaw changed from the prior `fetched=20, human_required=17` shape to `fetched=16, human_required=12`, and the overall privacy queue dropped from 50 to 45.

Remaining gaps:

- The remaining trusted remote blockers are mostly concrete host/path/account/private operational details. They should be abstracted or triaged, not auto-released by the dreaming filter.
- Codex and codex-cliproxyapi still have many oversize sessions; M25 needs the later pre-summary/session-selection pass to improve useful Codex lesson coverage.
