## Smoke Findings

- The run was intentionally degraded and did not call the LLM lesson extractor: `ai_distill.mode=disabled`, `lessons.ai_lessons=0`, and `ai_distill.warnings` includes `ai_distill_disabled`.
- Source coverage was real and broad: the lesson report selected 329 spans from 871 source items, including local OpenClaw, Codex, and trusted remote OpenClaw staged material.
- The lesson layer produced 22 deterministic lessons before dedupe/state routing. The final visible states in the daily report were `active_personal=1`, `wiki_ready=0`, `skill_ready=2`, `human_required=0`, and `rejected=0`.
- The useful visible lessons were:
  - private-route remote access as `active_personal`;
  - Slack raw user id boundary/redaction as `skill_ready`;
  - ACK before slow tool/network/dispatch work as `skill_ready`.
- Wiki proposal candidates were generated from lesson evidence and include M25 sections for applicability, negative cases, agent use, related pages, and span provenance.
- The generated HTML site exposes M25 lesson counts, top lesson summaries, pending wiki candidates, pending skill candidates, review reasons, and graph links. `dist/graph.json` contains 8 nodes, 29 links, and no broken links/orphans/duplicates.
- Stable PB pages are visible in graph/site source ids through `stable_kb:*` entries. GBrain and AgentMemory sidecars were not promotion evidence in this run; the authority contract explicitly ranks them below stable PB pages, promoted skills, and active personal lessons.

## Quality Gaps Before Full Daily

- This smoke does not prove production LLM lesson extraction quality because it used `--no-ai`. A follow-up AI smoke must run with a finite uncached budget and verify cache reuse.
- `wiki_ready=0` in this run. The wiki candidates were built from `skill_ready` lesson evidence and stable pages, not from a final wiki-ready lesson state.
- AgentMemory health failed with `agentmemory_health_failed: fetch failed`, so AgentMemory sidecar import/export behavior was not exercised.
- GBrain sidecar/export behavior was not exercised because there were no changed stable PB pages and no publish path was requested.
- Skill candidates still require human review. One candidate had `skill_shape_invalid:malformed_procedure_heading`; another had semantic review `revise` with a truncated procedure sentence.
- The review queue remains large because privacy triage reported 58 human-required exceptions. This is expected for private runtime material, but it blocks some potentially useful evidence until triaged.
- The current span provenance is useful for audit, but review surfaces must keep raw private excerpts hidden by default and should avoid making noisy candidate fragments the first user-facing content.

## Outcome

M25 real smoke is acceptable as a bounded degraded smoke: it proves memory-first ingestion, deterministic lesson routing, wiki/skill candidate generation, HTML visibility, privacy gating, and authority ranking. It is not sufficient as the final production quality proof. The next validation should be a small AI-enabled daily run with cached GLM extraction and sidecar health checks.
